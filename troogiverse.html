TRoOGiverse — Rules of Origin Analytics Tool

An interactive Shiny application that makes thousands of complex, multi-condition trade rule sets searchable and explorable by anyone — translating highly technical regulatory logic into a clean, fast, user-friendly interface.

Built with R Shiny, tidyverse, dplyr
Type Interactive Reference Tool
Data scope Thousands of product-country rules
Audience Technical & non-technical users

Overview

Rules of Origin are among the most technically complex elements of international trade agreements — each product category can have different qualifying conditions depending on the trading partner, the type of transformation applied, and the specific agreement in force. Navigating this manually is slow and error-prone, especially for non-specialist users.

TRoOGiverse solves this by structuring the entire rule set into a queryable, interactive database and wrapping it in a clean Shiny interface. Users can search by product code, country, or rule type and instantly see the relevant conditions — without needing to read through pages of legal text.

What it does

  • Fast search across thousands of rules

    Users can search by HS product code, country, or rule type and instantly retrieve the relevant qualifying conditions — no manual lookup required.

  • Multi-dimension filtering

    Filter chips allow users to narrow results by trading partner, product chapter, and rule type simultaneously — all updates happen reactively without page reloads.

  • Structured data modelling

    The underlying rule logic — including nested conditions, exceptions, and alternative qualifying criteria — was structured into a relational data model that enables consistent querying across all rule types.

  • Accessible to non-technical users

    The interface was designed so that non-specialists — trade operations teams, procurement staff, compliance teams — can use the tool independently without needing analyst support for every query.

  • Performance on high-cardinality data

    With thousands of unique product-country combinations, the app was built with performance front-of-mind — using efficient data structures and reactive caching to keep response times fast under real usage.

Interface design

A core design challenge was making a very large, structured dataset feel simple to navigate. The solution was a search-first interface — users start with a text query and progressively narrow results using filter chips, rather than facing a wall of dropdowns upfront.

Technical challenges

The biggest challenge was structuring the underlying rule data. Rules of Origin conditions are not uniform — different products have different rule types (Wholly Obtained, Change of Tariff Heading, Specific Process, etc.), some have multiple qualifying alternatives, and some have exceptions that only apply under certain conditions. Modelling all of this in a consistent, queryable structure required careful data design before any UI work could begin.

A secondary challenge was ensuring the app remained fast as the number of product-country combinations scaled. This was addressed through pre-filtering at the data layer and optimised reactive expressions that avoid redundant recalculation on every user interaction.
