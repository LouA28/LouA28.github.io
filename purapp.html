Import Preference Utilisation Rate App

An end-to-end interactive analytics application built in R Shiny — combining an automated ETL pipeline, geospatial mapping, time-series visualisation, and downloadable reporting on millions of rows of structured import data.

Built with R Shiny, ggplot2, Leaflet, SQL
Type Interactive Dashboard
Data scale Millions of rows
Users 50+ stakeholders

Overview

This application was designed to make large, complex import datasets explorable and actionable for a wide range of users — from technical analysts to senior non-technical stakeholders. The core challenge was taking raw, high-volume structured data and turning it into something interactive, fast, and genuinely useful.

The app integrates an automated data pipeline that extracts, cleans, and structures the data on each refresh — meaning users always see up-to-date outputs without any manual intervention. From there, all analysis and visualisation happens in real time inside the Shiny application.

What it does

  • Automated ETL pipeline

    Scripts automatically extract, transform, and load raw import data using R, dplyr, and data.table — handling missing values, schema inconsistencies, and large file sizes without manual intervention.

  • Geospatial choropleth mapping

    Interactive country-level maps built with Leaflet and sf, allowing users to visually explore geographic patterns at a glance and drill into specific regions dynamically.

  • Time-series visualisation

    Trend charts built in ggplot2 and Plotly showing how utilisation rates evolve over time across HS product sections and trading partners, with hover tooltips and zoom functionality.

  • Dynamic filtering and user controls

    Users can filter by country, product section, time period, and partner — all controls update every chart and map in real time using Shiny's reactive programming model.

  • Downloadable reports

    One-click export of filtered data and charts as formatted reports, allowing stakeholders to take outputs directly into briefings and presentations.

Pipeline architecture

The application follows a clean three-stage pipeline — data ingestion and transformation, then delivery through an interactive front end with multiple output types.

Technical challenges

The main engineering challenges were around performance — the underlying dataset is large and multi-dimensional, meaning naive implementations would result in slow load times and unresponsive filtering. This was solved through a combination of pre-aggregation during the ETL stage, efficient data structures (data.table over base R), and careful use of Shiny's reactive programming to minimise unnecessary recalculation.

Handling missing and inconsistent data across multiple source files was also a significant wrangling challenge, requiring robust cleaning logic that could account for varying schemas and reporting periods without breaking downstream visualisations.
