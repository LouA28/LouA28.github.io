PUR App — Trade Analytics Dashboard

A live R Shiny application processing millions of rows of structured data, used by 50+ stakeholders. Combines automatic data processing with interactive maps, time-series charts, and one-click reporting.

Built with R Shiny, Leaflet, ggplot2, Plotly, SQL
Type Interactive Dashboard
Data scale Millions of rows
Users 50+ stakeholders

Overview

This application was built to make a large, complex dataset easy to explore for a wide range of users, from technical analysts to senior colleagues with no data background. The goal was to take raw data that previously required manual work to interpret and turn it into something fast, interactive, and immediately useful.

The app automatically pulls in and processes fresh data on each refresh, so users always see up to date outputs without needing to ask anyone to run anything. All the charts, maps, and tables update in real time as users interact with the filters.

What it does

  • Automatic data processing

    The app pulls in raw data, cleans it, and structures it automatically using R, dplyr, and data.table. Missing values and file inconsistencies are handled without any manual steps needed each time.

  • Interactive country maps

    Country-level maps built with Leaflet and sf let users see geographic patterns at a glance, zoom into specific regions, and see values update as they apply filters.

  • Time-series and treemap charts

    Trend charts in ggplot2 and Plotly show how values change over time across different product groups and countries, with hover tooltips and zoom. Treemap visualisations built with treemapify show the relative size of different categories at a glance.

  • Filters and interactive tables

    Users can filter by country, product, time period, and partner. Every chart and map updates instantly. Interactive tables built with DT allow users to sort, search, and explore the underlying data directly in the app.

  • One-click report downloads

    Users can download formatted Excel reports of any filtered view with a single click. Reports are generated via openxlsx and used regularly by 50+ stakeholders to take outputs directly into meetings and briefings.

How the data flows

Data comes in as raw files, gets cleaned and structured automatically, and then feeds into the Shiny app where users can explore it through maps, charts, and tables, or download it as a report.

Challenges

The biggest challenge was keeping the app fast. The dataset is large and has many dimensions, so without careful design it would have been slow and unresponsive. This was solved by pre-processing and summarising data before it reaches the dashboard, using efficient R packages like data.table, and making sure only the parts of the app that need to update actually do so when a filter changes.

The raw data also came with quality issues, including missing values, inconsistent formats, and files with different structures across time periods. Building cleaning logic that could handle all of these automatically, without breaking anything downstream, was a significant part of the work.
