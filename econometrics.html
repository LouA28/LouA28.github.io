Econometric Modelling: Agricultural Import Drivers

A Python-based research project using statistical modelling on 512 observations of UK import data to identify what drives firms to claim preferential tariffs under the UK-EU Trade and Cooperation Agreement.

Built with Python, statsmodels, pandas
Data HMRC customs records, MacMap
Observations 512 product-country combinations
Published September 2025

Overview

This project used Python to analyse what factors influence whether firms claim preferential tariffs available under the UK-EU trade agreement. Working with 2022 import data across 512 product and country combinations, the goal was to build statistical models that could identify the key drivers and quantify their relative importance.

The dataset was built by combining three sources: HMRC customs records, tariff schedules from MacMap, and a rules of origin restrictiveness index. After cleaning and merging these, eight model specifications were run and compared to make sure the findings were consistent and reliable.

What the project involved

  • Data preparation across multiple sources

    Built the dataset in Python by combining HMRC customs records, MacMap tariff schedules, and a restrictiveness index. Handled missing values, inconsistent formats, and schema differences across sources before any modelling could begin.

  • Eight model specifications compared

    Rather than relying on a single model, eight versions were built using two different model types, starting simple and progressively adding controls. Comparing outputs across all eight gave confidence that the results were consistent rather than driven by any one modelling choice.

  • Results visualised and written up

    Findings were visualised using Matplotlib and written up as a formal research paper with methodology, results, and discussion sections. The paper was presented to senior colleagues and used to inform thinking on how well the trade agreement was working in practice.

Challenges

The main technical challenge was the shape of the data. The outcome variable is a proportion between 0 and 1, with many observations sitting at the extremes, either 0% or 100% utilisation. Standard regression does not handle this well, so two specialist model types were chosen and tested against each other to ensure the results were sound.

Merging three separate data sources with different structures, coverage periods, and product classification systems was also a significant wrangling task, requiring careful checking at each stage to make sure nothing was lost or mismatched in the merge.
